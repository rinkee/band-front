
✅ Always do
- 기능 추가 전 기존에 만들어둔 기능이 이미 있는지 검색
- 만들어진 기능이 있다면 최대한 활용해서 최적화
- 인덱스 관련 참고는 docs/supabase_indexes.md 우선 참고
- 유저의 문제점의 원인을 설명
- 유저가 질문임 이라고 물어본 경우는 질문에 대답만 할것.
- 유저가 계획을 짜보자 라고 물어본경우는 계획만 같이 논의할것.
- 1억명의 유저가 사용하는 서비스라고 생각하고 효율적이고 여럿이 사용해도 문제 없는 방향으로 코딩할것
⚠️ Ask first
- DB스키마 변경
- supabase fuction 업데이트

🚫 Never do
-하드코딩
- 비효율적인 코딩
- 기존 코드와 충돌되는 수정
- 유저가 질문임 이라고 물어봤는데 멋대로 코드수정 금지
- 유저가 계획을 짜보자 했는데 멋대로 코드 수정 금지 
- 디비 데이터 변경은 절대 금지
- 특히 프로덕션 디비는 절대 절대 직접 변경 금지
- Don't touch production DB
